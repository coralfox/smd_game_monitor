## 四、策略管理

策略是监控器的核心。每个策略定义了一套 "当出现某种卡死时，执行什么动作" 的规则。

### 4.1 策略列表

<img src="E:\云\Python\pythonProject\game_monitor\SMD游戏监控程序使用说明.assets\image-20260702123845861.png" alt="image-20260702123845861" style="zoom:67%;" />

### 4.2 添加/编辑策略

点击 "添加策略" 或选中策略后点击 "编辑策略"，弹出策略编辑窗口：

<img src="E:\云\Python\pythonProject\game_monitor\SMD游戏监控程序使用说明.assets\image-20260702123926307.png" alt="image-20260702123926307" style="zoom:67%;" />

**字段说明**

| 字段 | 说明 |
|------|------|
| 策略 ID | 唯一标识，不能重复 |
| 策略名称 | 显示用的名称 |
| 匹配文本 | OCR 结果中包含这些文本时，该策略生效。留空表示匹配所有 |
| 排除文本 | OCR 结果中 **不** 包含这些文本时，该策略生效 |
| 匹配卡死类型 | 单一卡死 / 交替卡死 / 任意 |
| 检测阈值 | 在统计窗口内该事件出现多少次才判定卡死 |
| 占比 | 该事件占样本总数的比例达到多少才判定卡死 |

### 4.3 卡死类型详解

**单一卡死**

在统计窗口内，某个事件反复出现，次数和占比都达到阈值。例如：

```
[当前事件-58-移动] 出现 32 次 (占比 80.0%, 阈值 30/80%)
```

**交替卡死**

在统计窗口内，两个事件交替出现，合计次数和占比都达到阈值。例如：

```
[当前事件-36-移动]x20  [当前事件-35-按键]x18 (合计占比 97.6%, 阈值 40/85%)
```

### 4.4 动作类型

策略触发后执行的动作，支持多种类型：

| 动作类型 | 参数 | 说明 |
|----------|------|------|
| `key_press` | `key`, `presses` | 按下指定按键，如 `"p"` 按 1 次 |
| `mouse_click` | `button`, `x`, `y` | 在指定坐标点击鼠标 |
| `mouse_move` | `x`, `y` | 移动鼠标到指定坐标 |
| `text_input` | `text` | 输入一段文字 |
| `wait` | `duration` | 等待指定秒数 |
| `delay` | `seconds` | 延迟指定秒数 |
| `log` | `message` | 输出日志 |
| `screenshot` | (无) | 截取游戏窗口全屏，保存到 `异常/` 目录 |

**screenshot 动作示例**

```json
{"type": "screenshot"}
```

触发后会自动截取整个游戏窗口，文件命名为 `策略名_20260702_071828.png`，保存到程序目录下的 `异常/` 文件夹中。

## 五、使用流程示例

### 示例 1：单一卡死处理（按 P 键 * 2）

大部分脚本卡死时，都是卡在某一个事件，按 `P` 键可以停止脚本，再按 `P` 就可以最近点正常继续。

**步骤**

1. 选择游戏窗口，设置监控区域（脚本状态文字所在位置）, 确认能够识别到所有信息

   ![image-20260702124530832](E:\云\Python\pythonProject\game_monitor\SMD游戏监控程序使用说明.assets\image-20260702124530832.png)

   <img src="E:\云\Python\pythonProject\game_monitor\SMD游戏监控程序使用说明.assets\image-20260702124718473.png" alt="image-20260702124718473" style="zoom: 67%;" />

2. 确认监控参数设置正确, pushplus 或邮件配置正确 **(可逃过)**

3. 在 "策略管理" 中确认有 "单一/交替卡死处理" 策略 **(可逃过)**

4. 点击 "**开始监控**" **(F8 启动, F10 停止)**

**工作原理**

- 每次 OCR 识别到文字时，检查是否包含 "当前事件 移动"
- 如果 **包含** "当前事件 移动"，则给 **"单一卡死处理"** 策略添加一个样本
- 当样本积累到阈值（30 次，占比 80%）时，触发策略
- 策略执行：按 **P** 键 **2** 次尝试恢复正常

**举例说明**

> 卡 **在当前事件-34-移动** 1 分钟了, 程序每隔 1 秒截图识别, 然后 60 秒内, 发现这一事件在所有移动事件里达到 30 次了, 并且占比 80%了, 说明卡在这个移动了，一直无法到达。**单一卡死处理** 触发，激活游戏窗口，发送 **2 次 P**，让原力重启脚本。

2. 确认监控参数设置正确, pushplus 或邮件配置正确 **(可逃过)**

3. 在 "策略管理" 中确认有 "交替卡死处理" 策略 **(可逃过)**

### 示例 3：路径错误专用处理

![image-20260702133020363](E:\云\Python\pythonProject\game_monitor\SMD游戏监控程序使用说明.assets\image-20260702133020363.png)

**步骤**

1. 选择游戏窗口，设置监控区域（脚本状态文字所在位置）, 确认能够识别到所有信息

## 十、常见问题

**Q: OCR 识别不准确怎么办？**

A: 程序内置了 OCR 文本标准化，会自动处理以下情况：
- 下划线 `_` → 连字符 `-`
- 等号 `=` → 连字符 `-`
- 全角符号 → 半角符号
- 数字与汉字之间缺少连字符时自动补全，如 `当前事件57移动` → `当前事件-57-移动`

如果仍不准确，可以调整监控区域的大小和位置，确保文字清晰。

**Q: 策略一直不触发怎么办？**

A: 检查以下几点：
1. 监控区域是否正确，OCR 能否识别到文字
2. 检查阈值和占比是否设置过高
3. 检查 "匹配文本" 和 "排除文本" 是否正确
4. 查看日志中的统计信息

**Q: 误触发太频繁怎么办？**

A: 可以：
1. 提高阈值（如从 30 提高到 50）
2. 提高占比（如从 80% 提高到 90%）
3. 增加触发冷却时间
4. 添加更精确的 "匹配文本" 或 "排除文本"

**Q: 如何备份配置？**

A: 配置文件都在 `configs/` 文件夹中，直接复制 `.json` 文件即可备份。也可以点击 "另存为" 创建多个配置副本。
